# claude-usage-widget

Native macOS desktop widget — a **faithful Clawdmeter clone** showing live Claude Code **subscription** usage. Tauri v2 (Rust + vanilla web UI).

Features: Current (5h) + Weekly (7d) usage % as **alert-zone bars** — color encodes *state only* (safe / amber≥75 / red≥90), magnitude is the bar *length* (NN/g preattentive; the bound number also takes the zone hue for colorblind redundancy) + reset timers (5h also shows clock time); **binding-constraint aware** — risk/ETA/footer consider *both* 5h and weekly (either limit blocks; weekly is the multi-day lockout people get blindsided by) and an accent pill marks whichever is nearer its cap (only under pressure ≥50%); **ETA-to-limit** ("limit in Xh Ym" from the utilization slope) + **trend** (▲%/h) + **blocked takeover** (whole-card red veil + sleeping Clawd when `status`=rejected) + **80/95% + weekly-90% macOS notifications**; animated Clawd mascot (13 ClaudePix animations, **mood driven by burn-rate/velocity**, not absolute % — true to upstream); **⤢ expand** → detail panel led by the **ETA headline**, then cost/burn/projection/models (ccusage pseudo-model `<synthetic>` filtered out)/tokens/**cache-hit %** via ccusage; **click mascot → creature mode** (big idle Clawd, click cycles figure, ↩ back); **📌 pin → PiP mode** (all Spaces + over fullscreen + on top); **⚙ settings** (theme **dark/light**, **glass effect** on/off [native macOS vibrancy, default off], notifications on/off, **plan** Pro/Max5×/Max20× manual, **cost comparison**: subscription $/mo vs API-equivalent block + month via `ccusage monthly`); **✕** (hover) closes; whole card draggable, **remembers position**; Clawd app icon.

## Data — two sources
- **Primary: rate-limit % (`get_usage`)** — reads the Claude Code OAuth token from the macOS **Keychain** (`security find-generic-password -s "Claude Code-credentials"`; fallback `~/.claude/.credentials.json`), POSTs one minimal `/v1/messages` (`anthropic-beta: oauth-2025-04-20`, haiku, max_tokens 1), reads response headers `anthropic-ratelimit-unified-{5h,7d}-{utilization,reset}` + `-5h-status`. **Subscription auth, not API-billed.** Pure header parser is unit-tested. Polls every 30s.
- **Secondary: cost (`get_cost`)** — runs `ccusage@14 blocks --active --json` → `costUSD`/`burnRate.costPerHour`/`projection.totalCost`/`models`/`tokenCounts`. Only polled while the info panel is open. (`costUSD` needs explicit serde `rename` — uppercase acronym.) **Pinned to `@14`**: v15+ ships a native binary that crashes with a nix-libiconv `dyld` error on this Mac; v14 is the last pure-JS release.

## Build / run
- Prereqs: **Rust** (rustup, `cargo 1.96`; non-login shells: `source "$HOME/.cargo/env"`), Xcode CLT, Node v22, ccusage reachable via `npx`.
- Dev: `npm run tauri dev` · Build: `npm run tauri build --bundles app` (→ `.app` in `src-tauri/target/release/bundle/macos/`; `bundle.targets` = app/appimage, **no `.dmg`**).
- Test: `cargo test --manifest-path src-tauri/Cargo.toml` — 17 unit tests (rate-limit parser, token extraction, ccusage cost/month parsers, OAuth PKCE) are the gate. **Gotcha:** a stale `target/build/tauri-*` cache can bake an old absolute path (`~/Desktop/Ton/...` from before the repo moved) → `failed to read plugin permissions`; fix with `rm -rf target/{debug,release}/build/tauri-*` (regenerates, no full clean). And `cmd | tail` masks cargo's exit code — check the real one.
- **Install / distribution**: ships as **source only** (public repo, but no installers — bundles Clawd/Anthropic assets). Users build from source: `npm install && npm run tauri build` then run the `.app` (macOS) / AppImage (Linux). **No install script** — people (rightly) don't run unknown `.sh`; the README leads with the transparent `tauri build` + an "ask your Claude Code to build it" path. `bundle.targets` = `["app","appimage"]` (no `.dmg`). CI is `.github/workflows/ci.yml` — `cargo test` gate only. Linux: macOS-only deps (`objc`, `tauri-nspanel`) are target-gated; token falls back to `~/.claude/.credentials.json`; PiP degrades to always-on-top + all-workspaces.
- Perf / leveza notes: Tauri commands are async (`spawn_blocking`) — blocking I/O on the main thread froze the UI every poll; ccusage cmd/token/HTTP-agent are cached (`OnceLock`); position saves throttled (1/s + flush on close); animations pre-rendered offscreen + JSON cached; polling decays 30s→2min when utilization is flat; **animation loop runs on `requestAnimationFrame`** (WebKit self-suspends it when the widget is occluded / off-Space / display-asleep — `setTimeout` kept waking the CPU and only paused on `document.hidden`, which a PiP window rarely is). **Memory ~150 MB is the WKWebView floor, not our content** (canvas backing stores are 20×20, DOM is tiny, no leaks) — don't chase it; idle CPU is ~0.1%. Leveza = energy, and the lever is the animation loop.
- Icon: render a Clawd frame to a 1024 PNG (PIL, see git history) → `npm run tauri icon <png>`.
- `CCUSAGE_CMD` / `CLAUDE_CODE_TOKEN` env overrides (the latter handy for headless testing without the Keychain).

### ⚠️ This dev machine's gotchas
- **iCloud-synced `~/Desktop`** corrupts the shell cwd inode mid-session → `getcwd`/`uv_cwd` EPERM for spawned binaries (cargo/npm/node/python). **Workaround: prefix build/run commands with `cd /` and use absolute paths** (e.g. `cd / && npm --prefix <abs> run tauri -- build`). A fresh terminal also resets it. Long-term fix: move the repo out of `~/Desktop`.
- **macOS PATH**: a Finder-launched `.app` only inherits `/usr/bin:/bin:...` — `usage.rs::resolve_program()`/`extra_node_dirs()` resolve `npx` to an absolute path (nvm + homebrew) so the bundled app finds node.
- **set_position is LOGICAL points**, not physical pixels — mixing physical on Retina hides the window off-screen. `lib.rs::top_right_pos` converts via `scale_factor`. Verify window placement with `CGWindowListCopyWindowInfo` (JXA): expect `X≈2264, onscreen=true`.
- First `.app` launch may prompt to allow Keychain access → **Allow**.
- **Window commands need capability permissions**: JS `setSize`/`startDragging`/`close`/etc. silently fail unless granted in `src-tauri/capabilities/default.json` — `core:default` does NOT include the mutating ones. We grant `core:window:allow-{set-size,set-always-on-top,close,start-dragging}`. (Drag and resize were both broken purely from missing perms.)
- **PiP mode** (`apply_pip` in `lib.rs`, macOS): NSWindow `collectionBehavior = CanJoinAllSpaces|FullScreenAuxiliary` (raw `objc`) + re-assert `set_always_on_top` (collectionBehavior resets the level; toggle off→on to force re-apply). **`CGWindowListCopyWindowInfo` reports `layer=0` for all-Spaces windows even though the real NSWindow `level` is 5** — don't trust CGWindowLayer here; read `[nsWindow level]` directly to verify. Also: this PiP NSPanel **doesn't show up in `CGWindowListCopyWindowInfo` by PID at all** — so I can't screenshot/locate it from the shell (Screen-Recording + Accessibility TCC are also denied). Prove the app is alive instead via its **TLS keep-alive to `api.anthropic.com:443`** (`lsof -nP -a -p <pid> -i`) = the poll loop ran = webview is up. If "not opening": likely **pin is OFF** (unpinned non-activating panel doesn't come to front → buried behind windows), not a crash.
- **Glass effect** (`set_glass`, macOS): native `NSVisualEffectMaterial::UnderWindowBackground` via `window-vibrancy` (the most translucent material — HudWindow washed the text out on a busy wallpaper; under-window read better and Ton picked it); the card CSS goes translucent (`[data-glass="on"]` → `--card-bg`/`--panel`) to reveal it. It renders dark-frosted in a dark-appearance window → white text stays legible. **The vibrancy view is a rectangle** — once the contentView is layer-backed its square corners poke out past the card's rounded corners, so we **always** clip `contentView.layer` to `cornerRadius 18 + masksToBounds` via objc — glass on *or* off (the crate's `radius` arg wasn't reliably clipped on the NSPanel). Default **off** (`cuw-glass`). For convincing glass: low card tint (~5–10%) + a specular sheen gradient in the card *background* + bright top / dark bottom inset rims — multi-factor, not just opacity. **localStorage is UTF-16LE** if you ever poke the sqlite directly (`x'3100'` = `"1"`, not `x'31'`).

## Architecture
- `src-tauri/src/usage.rs` — `get_usage` (rate-limit %, pure `parse_rate_limit`) + `get_cost` (ccusage, pure `parse_cost`) + Keychain token read + PATH resolution. All parsers unit-tested against `tests/fixtures/{active,idle}.json`.
- `src-tauri/src/lib.rs` — window config + `top_right_pos` (logical, origin-monitor) + **position persistence** (`window.json` in `app_config_dir`, saved on `WindowEvent::Moved`, restored in `setup`) + `set_pinned`/`apply_pip` (PiP) + **`set_glass`** (optional native vibrancy via `window-vibrancy` + contentView-layer corner-clip).
- `src/` (vanilla HTML/CSS/JS) — view state machine `compact|info|creature|settings|connect` (`data-view`), `data-state` overlays, canvas **animation engine** (palette-indexed 20×20 frames from `src/assets/animations/*.json`, **driven by `requestAnimationFrame`** with a per-frame `hold` gate; mascot **mood = burn-rate**), `setSize` on view change. Alert zones via `data-zone`; binding marker via `data-binding`; glass via `data-glass` (`--card-bg`/`--panel` translucent). Drag via `data-tauri-drag-region="deep"` (buttons block naturally; mascot canvas opts out with `="false"`). Test hooks `__cuwRender`/`__cuwRenderCost`.
- Assets: `src/assets/fonts/` (Tiempos, StyreneB), `src/assets/animations/` (13 Clawd JSON) — from upstream Clawdmeter (private use).
